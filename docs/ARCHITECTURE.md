### Multi-Tenant Routing

The application serves multiple domains from a single codebase:

| Domain | Purpose | Routes |
|--------|---------|--------|
| `yeoskin.fr` | Creator portal + Auth | `/c/*`, `/auth/*` |

## Key Design Patterns

### 1. Authentication Middleware

All protected API routes use a shared auth middleware:

```typescript
// src/lib/auth-middleware.ts
export async function verifyAdminAuth(req: NextRequest): Promise<AuthResult> {
  // 1. Extract token from Authorization header
  // 2. Verify with Supabase Auth
  // 3. Check admin_profiles table
  // 4. Return auth result with role
}

// Usage in API routes:
const auth = await verifyAdminAuth(request)
if (!auth.authenticated) {
  return unauthorizedResponse(auth.error)
}
```

### 2. SWR Data Fetching

Optimized data fetching with smart caching:

```typescript
// src/hooks/useSWROptimized.js
const swrConfig = {
  revalidateOnFocus: false,
  dedupingInterval: 5000,
  errorRetryCount: 3,
}

// Different refresh intervals per data type
useDashboardStats()    // 5 min
useCreators()          // 2 min
useBatches()           // 1 min
```

### 3. Financial State Machine

Payouts follow a strict state machine:

```
Commission States:
  pending â†’ locked â†’ payable â†’ paid
                  â†˜ canceled

Batch States:
  draft â†’ approved â†’ executing â†’ sent â†’ paid

```

### 4. Idempotency Pattern

Payment operations use idempotency keys:

```typescript
// Prevent duplicate payments
const idempotencyKey = `checkout:${creatorId}:${orderId}`
const existing = await checkIdempotencyKey(idempotencyKey)
if (existing) return existing.response
```

## Performance Considerations

### Caching Strategy

| Data Type | Cache Duration | Strategy |
|-----------|----------------|----------|
| Dashboard stats | 5 min | SWR stale-while-revalidate |
| Creator list | 2 min | SWR with optimistic updates |
| Batch data | 1 min | SWR with real-time fallback |
| Static content | 24 hours | ISR (if implemented) |

### Database Optimization

- Indexed columns: `email`, `status`, `created_at`, `creator_id`
- Materialized views: `creator_balances`, `financial_summary`
- Connection pooling: Supabase PgBouncer
